# 비개발자도 가능한 AI 앱 제작 회고 (Notion 정리본)

작성일: 2026-03-23  
프로젝트: 레슨 예약 서비스 만들기

---

## 1) 한 줄 요약
이번 프로젝트는 AI를 이용해 “기능만 빨리 만드는 것”이 아니라, **실제로 사람들이 쓰기 편한 서비스로 다듬는 과정**에 집중했다.

## 2) 왜 이 글을 쓰는가
많은 사람이 AI를 쓰면 코딩을 잘해야 한다고 생각한다.  
하지만 실제로는 비개발자도 충분히 방향을 잡고, AI와 협업해서 서비스 완성도를 끌어올릴 수 있다.

이 문서는 그 과정을 비개발자 눈높이로 정리한 기록이다.

## 3) 내가 한 일 (비개발자 역할)
- 사용자의 입장에서 불편한 지점을 말해줌
- “개발자 화면 말고 사용자 화면 중심으로” 같은 방향을 계속 제시함
- 최종적으로 보기 쉽고, 쓰기 쉽고, 이해하기 쉬운지 확인함

핵심은 **직접 코드를 다 짜는 것**이 아니라,  
**무엇을 고쳐야 하는지 정확히 요청하는 것**이었다.

## 4) AI가 도와준 일
- 실행이 안 되던 환경 복구
- 화면에서 불필요한 개발자용 요소 제거
- 모바일에서 누르기/보기 편하게 UI 정리
- 테스트와 점검 항목 자동 정리
- 회의/인수인계를 위한 문서 작성

즉, AI는 단순 답변 도구가 아니라 **작업 파트너**처럼 움직였다.

## 5) 이번에 좋아진 점
### 5-1) 사용자 중심 화면
- 개발자 로그 화면 제거
- 어려운 표현을 쉬운 표현으로 변경
- 성공/실패 안내를 한눈에 보이게 정리

### 5-2) 모바일 사용성
- 작은 화면에서도 버튼 누르기 편하게 개선
- 메뉴/표/팝업이 겹치거나 잘리지 않게 정리

### 5-3) 운영 준비
- 점검 체크리스트 생성
- 디자인 검수 시 주의사항 문서화
- 화면 비교용 스냅샷 보관

## 6) 느낀 점 (비개발자 관점)
AI를 잘 쓰는 핵심은 기술 지식보다 **요청하는 방식**에 가깝다.

좋은 요청의 기준:
1. 목적을 분명히 말하기 (예: 사용자 화면 중심)
2. 하지 말아야 할 것을 같이 말하기 (예: 기능 추가 금지)
3. 완료 기준을 말하기 (예: 모바일 최적화 + 체크리스트 문서)

이 3가지만 지켜도 결과 퀄리티가 크게 달라진다.

## 7) 같은 방식으로 해보고 싶은 사람에게
비개발자라면 아래처럼 요청하면 된다.

```text
목표: 사용자 입장에서 보기/쓰기 편한 서비스로 정리해줘.
제약: 기능 추가는 하지 말고, 기존 기능 품질만 올려줘.
우선순위: 모바일 사용성 > 쉬운 문구 > 화면 정돈.
완료조건: 점검 체크리스트와 결과 문서까지 남겨줘.
```

이 템플릿만으로도 AI 협업의 결과가 훨씬 안정적이다.

# 블로그용 쉬운 버전 (초안)

## 제목 후보
1. 코딩 몰라도 AI로 서비스 만들 수 있을까? 직접 해본 후기
2. 비개발자가 AI와 함께 앱 완성도를 올린 방법
3. 기능 추가 없이도 서비스가 좋아지는 AI 협업 방식

## 오프닝
처음에는 “AI로 앱 만든다”는 말이 개발자 이야기처럼 들렸다.  
그런데 직접 해보니 중요한 건 코딩 실력보다, 사용자 관점에서 방향을 잘 주는 일이었다.  
이번에는 기능을 늘리지 않고도 화면과 사용성을 크게 개선했고, 그 과정을 정리해봤다.

## 본문 구성 추천
1. 비개발자인 내가 어떤 역할을 했는지
2. AI가 실제로 대신 해준 일
3. 결과가 좋아진 포인트(사용성/모바일/정리 문서)
4. 시행착오와 배운 점
5. 누구나 바로 써볼 수 있는 요청 템플릿

## 마무리
AI 시대에 비개발자가 할 일은 줄어든 게 아니라 더 명확해졌다.  
기술 구현은 AI가 많이 도와주고, 사람은 사용자 관점과 우선순위를 잡는다.  
이 조합이 생각보다 강력했다.
