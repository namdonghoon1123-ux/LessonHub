## 추가 인수인계 (블럭 단위 운영 전환)
- 핵심 정책 변경
  - 학생 예약은 `teacher_profiles.lesson_duration_min`이 아니라 "교사가 생성한 블럭 길이"를 따름
  - 예약 생성 시 슬롯의 `duration_min`과 요청 `duration_min`이 다르면 `duration_min_mismatch` 반환
- 데이터 모델
  - `one_time_availabilities` 테이블 추가 (`006_one_time_availabilities.sql`)
  - 단일 생성은 이 테이블에 1건 저장하고, 과거처럼 예외를 대량 생성하지 않음
- 슬롯 계산
  - `/api/v1/teachers/:teacherId/slots`는 주간 반복 + one-time 블럭을 합쳐 블럭 단위로 반환
  - 예외(`availability_exceptions`) 및 예약 점유(`PENDING`,`BOOKED`)를 동일하게 필터링
- 예외 UX 변경
  - 예외 입력은 블럭 단위(시작/종료 필수)로 단순화
  - 종일 체크박스 기반 대량 예외 생성 경로는 제거
- 초대/캘린더
  - 교사 초대 링크 복사 버튼: `/student-calendar.html?teacher=<id>`
  - 학생 페이지는 `teacher` 쿼리 파라미터로 초기 교사 자동 선택
  - 한국 공휴일(KR) 라벨/셀 하이라이트 표시 추가

## 추가 인수인계 (게스트 비로그인 + 코멘트 분리 + 설정파일)
- 설정파일 경로
  - 저장소: `backend/config/app.config.json`
  - 컨테이너 런타임: `/app/config/app.config.json`
  - 커스텀 파일 지정: `APP_CONFIG_FILE` 환경변수
- 현재 설정 키
  - `guest.publicAccessDays`
  - `guest.dataRetentionDays`
  - `guest.cancelReasonRequired`
  - `guest.rateLimit.windowMs`, `guest.rateLimit.max`
  - `guest.pin.maxFailedAttempts`, `guest.pin.lockoutMinutes`
  - `booking.comments.requireTeacherPrivateOnComplete`
  - `booking.comments.requireStudentCommentOnComplete`
  - `schedulers.autoCompletePollMs`, `schedulers.guestRetentionPollMs`
- 코멘트 분리 정책
  - 교사용 내부 메모: `bookings.teacher_private_comment`
  - 학생 전달 코멘트: `bookings.student_comment`
  - 완료 API: `POST /api/v1/teachers/me/bookings/:id/complete`
    - 입력: `teacher_private_comment`, `student_comment`
    - 기본 설정상 두 필드 모두 필수
- 게스트 비로그인 흐름
  - 생성: `POST /api/v1/public/bookings` (`student_name`, `phone`, `pin`)
  - 조회: `POST /api/v1/public/bookings/lookup` (`phone`, `pin`)
  - 취소: `POST /api/v1/public/bookings/:id/cancel` (`phone`, `pin`, `reason`)
  - 링크취소: `POST /api/v1/public/bookings/:id/cancel-by-token` (`token`, `reason`)
  - PIN 연속 실패 시 락아웃(설정값 기반)
  - 교사 PIN 초기화: `POST /api/v1/teachers/me/guest-students/:id/reset-pin`
- 마이그레이션 최신 상태
  - `001` ~ `009` 적용 필요
  - 신규: `009_guest_pin_lockout_and_comment_split.sql`
